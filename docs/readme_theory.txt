Readme - это главный файл моего репозитория на GitHub - Его видит каждый, кто заходит в проект 

Зачем он нужен? 
1. Рассказать, что делает проект 
2. Показать как установить и запустить
3. Продемонстрировать применяемые технологии
4. Привлечь нужных людей

Формат: Markdown (.md) - легкий язык разметки 

Структура readme: 
1. Заголовок - название проекта 
2. Описание - о чем проект, 2 - 3 предложения 
3. Технологии - список применяемых технологий 
4. Демо/скриншоты - 2, 3 картинки работающего приложения 
5. Установка - пошаговая инструкция 
6. Запуск - команды для запуска 
7. Пример использования - скриншот предсказания 
8. Автор - имя + ссылка на GitHub
9. Лицензия - MIT (или другая )

Заголовок 1 - # Текст - # Job Analyzer
Заголовок 2 - ## Текст - ## Установка
