Зачем нужен git? - git это система контроля версий, которая позволяет сохранять историю изменений в коде
1. История изменений и откаты в коде 
2. Совместная разработка в команде 
3. Ветвтление: чтобы не затрагивать основной код, можно сделать ветку 
4. Безопасность и хранение: на случай потери есть копии 
5. Документация процесса изменений 
6. Можно "откатить" код 

Что такое GitHub? - это облачное хранилище для Git(репозиториев)
1. Удаленное хранилище 
2. Комндная работа 
3. Портфолио 
4. Open Source - "социальная сеть" разработчиков 
5. Автоматизацаия (CI/CD): 
5.1 CI - continuous Integration - когда программист сохраняет свой коммит(код) система автоматически: 
-собирает проект
-Запускает тесты на новый функционал 
-Анализирует качество кода на соответствие стандартам 
5.2 CD - continuous Delivery - если все тесты пройдены, код автоматически готовиться к выпуску. Отправляет в предпродакшен,
где его могут проверить тестировщики 
5.2.1 CD - continuous Deployment - непрервные развертывание - когда тесты пройдены и сразу попадают к реальным пользователям 



Жизненный цикл файлов git: 
1.Untracked (Неотслеживаемые) - это новые файлы, которые только что созданы. GIT видит их в папке, но не следит за изменениями
внутри их 
2. Tracked (Отслеживаемые) - это файлы которые уже известны git 
- Unmodified (Неизмененные) - файлы, которые идентичны последней сохраненной версии. 
- Modified (Измененные) - кто -то внес правки в файл, но еще не подготовил к сохранению. 
Staged (Подготовленные) - файл находится в индексе 




Ключевые скрипты git: 

git config --global user.name "Имя" 
git config --global user.email "Почта" 

git init - инициализация репозитория 
git status - проверка статуса 

git add app.py - добавление файлов в индекс (промежуточный этап перед оконачетельным сохранением - индекс) 
git add src/               - вся папка
git add .                - все изменения 

git commit -m "feat: ..." коммит сохранение (снимок состояния файлов)

git branch branch-name - отдельная линия разработки 

git remote add origin URL - удалить репозиторий (GitHub)

git log - полная история 
git log --online - сокращенная история 


Работа с GitHub: 

git remote add origin https:// github.com/username/repo.git - добавление удаленного репозитория 

git branch -M main - используем принудительное переименование текущей ветки в main 

git push -u origin main  - отправка на GitHub

git pull origin main - скачивание с github



Дополнительные нужные команды: 

git clone - клонирует репозиторий c GitHub

cd .. - переход в папку 

python -m venv venv - создаем виртуальное окружение 

venv/Scripts/activate - активирует виртуальное окружение 

pip install -r requirements.txt - установка зависимостей библиотек 

python src/db/database.py - создание таблиц в БД 

python src/web/app.py - запуск сервера 





.gitignore - это текстовый файл, в котором указываем git, какие файлы и папки в проекте нужно игнорировать 
Не надо добавлять: 
-.env - файлы с ключами доступа к БД и API 
-*.pem, id_rsa — файлы ключей и сертификатов.
- config/database.yml — если там прописаны реальные пароли.

-node_modules/ (для JavaScript).
-venv/, .venv/, env/ (виртуальные окружения Python).
-vendor/ (для PHP/Composer).

-.DS_Store — (для macOS).
-Thumbs.db — (для Windows).
-.idea/, .vscode/ — настройки ваших редакторов кода (IntelliJ IDEA, VS Code).

-dist/, build/, out/ — папки с готовым скомпилированным кодом.
-*.log — файлы логов (журналы ошибок).
-*.exe, *.pyc, *.class — скомпилированные бинарные файлы.

- *.png - не добавлять фотографии
